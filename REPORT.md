# 분석 보고서: 소비재 기업 감사보수 결정요인 분석

## 1. 배경 및 목적

감사보수(Audit Fee)는 감사인의 노력·위험·전문성에 대한 대가로, 감사 실무와 회계학 연구에서
오랫동안 다뤄온 주제입니다. 선행연구들은 일반적으로 다음 요인들이 감사보수에 유의미한 영향을
미친다고 보고합니다.

- **기업 규모**: 자산·매출액이 클수록 감사 범위가 넓어져 보수가 증가
- **감사인 유형**: Big4(대형 회계법인) 감사 시 보수가 더 높은 경향 (품질 프리미엄)
- **재무위험**: 부채비율이 높거나 감사의견이 비적정일 경우 보수가 증가하는 경향

본 프로젝트는 DART(전자공시시스템) 데이터를 활용해 이 요인들을 실제로 데이터에서 확인하고,
머신러닝 회귀 모델로 감사보수를 예측하는 감사 분야 취업 포트폴리오입니다.

## 2. 데이터 개요

| 항목 | 내용 |
|---|---|
| 분석 대상 | 소비재(식품·음료·생활용품·화장품) 상장기업 12개사 |
| 분석 기간 | 2018 ~ 2023년 (6개년) |
| 표본 크기 | 72건 (12개사 x 6개년) |
| 변수 | 자산총계, 매출액, 부채총계, 자본총계, 업종, 감사인, 감사의견, 감사보수, 감사시간 |

> ⚠️ 감사보수·감사인 데이터는 DART 감사용역체결현황 API의 정확한 응답 필드를 이 프로젝트
> 작성 시점에 실시간으로 검증하지 못해, 선행연구의 결정요인(기업 규모, Big4 여부)이 반영되도록
> **가상으로 생성한 샘플 데이터**를 사용했습니다. 재무데이터(자산·매출·부채·자본)는 DART
> 재무제표 API 구조를 그대로 따른 수집 코드로 작성되었습니다.

## 3. 분석 방법론

1. 재무정보와 감사보수정보를 기업명·연도 기준으로 병합하여 통합 데이터셋 구성
2. 감사인·감사의견·업종 등 범주형 변수의 분포를 EDA로 확인
3. Big4 여부, 감사의견 적정 여부를 이진 변수로 피처 엔지니어링
4. 상관관계 분석으로 변수 간 관계 및 다중공선성 점검
5. 회귀모델(LinearRegression, RandomForest, GradientBoosting, XGBoost) 학습 및 성능 비교
6. 딥러닝(Keras) 모델의 학습 곡선을 통해 모델 학습 상태 점검

## 4. 다중공선성 문제 발견 및 해결 과정

분석 초기 상관관계 히트맵에서 `assets`(자산총계), `debt`(부채총계), `equity`(자본총계),
`revenue`(매출액) 간 상관계수가 0.79~0.98으로 매우 높게 나타났습니다.

원인을 확인해보니, `자산총계 = 부채총계 + 자본총계`라는 **회계항등식**으로 인해 `equity`가
`assets`와 `debt`만으로 완전히 결정되는 중복 변수였습니다.

```
assets - (debt + equity) ≈ 0  (반올림 오차 수준)
```

이 상태로 모델링했을 때 나타난 문제는 다음과 같습니다.

| 모델 | 문제 현상 |
|---|---|
| LinearRegression | 회귀계수가 ±10¹² 수준으로 비정상적으로 폭발 (다중공선성으로 인한 계수 불안정) |
| RandomForest | "기업 규모"라는 하나의 신호가 assets·debt·equity·revenue 4개 변수에 쪼개져, 각 변수의 개별 중요도가 5~7%로 과소평가됨 |

**해결**: `equity`를 완전히 제거하고, `assets`는 감사보수 선행연구에서 흔히 쓰이는 로그 규모
지표 `log_assets`로, `debt`는 절대금액 대신 `leverage`(부채비율 = 부채총계/자산총계)로
재정의했습니다. 그 결과 회귀계수가 정상 범위로 돌아왔고, `log_assets`의 개별 기여도(약 12%)도
명확하게 드러났습니다.

## 5. 주요 결과

### 5.1 상관관계 (감사보수 기준)

| 변수 | 상관계수 |
|---|---|
| is_big4 | 0.83 |
| equity (수정 전 assets 계열 중 최고) | 0.56 |
| log_assets | 0.52 |
| revenue | 0.52 |
| debt | 0.42 |
| is_clean_opinion | 0.04 |

### 5.2 RandomForest Feature Importance (수정 후)

| 변수 | 중요도 |
|---|---|
| is_big4 | 약 69% |
| log_assets | 약 12% |
| revenue | 약 8% |
| leverage | 약 5% |
| audit_hours | 약 4% |
| 기타(업종, 감사의견) | 약 2% |

### 5.3 모델 성능 비교

| 모델 | 특징 |
|---|---|
| LinearRegression | 다중공선성 해소 후 가장 안정적인 성능 |
| RandomForest | 비선형 관계 포착, feature importance 해석 용이 |
| GradientBoosting / XGBoost | 부스팅 기반, 소규모 표본에서는 과적합 위험 존재 |

(정확한 MAE/RMSE/R² 수치는 노트북 실행 결과를 참고하세요. 학습 시 `random_state=42`로
고정되어 있어 재현 가능합니다.)

### 5.4 딥러닝 모델 학습 곡선

Keras 기반 딥러닝 회귀 모델을 EarlyStopping(patience=10)과 함께 학습한 결과, validation loss가
일정 시점 이후 더 이상 개선되지 않아 조기 종료되었습니다. 이는 모델이 과대적합 없이 적절한
시점에 학습을 마쳤음을 의미합니다.

## 6. 결론

- **Big4 여부가 감사보수를 결정하는 압도적으로 지배적인 요인**으로 나타났으며, 이는 대형
  회계법인의 품질 프리미엄에 관한 기존 연구와 방향이 일치합니다.
- **기업 규모(log_assets)도 유의미한 양(+)의 영향**을 미치지만, 이번 표본에서는 Big4 여부에
  비해 상대적으로 작은 비중을 차지했습니다. 두 요인의 영향력을 동등하게 서술하는 것은 실제
  분석 결과를 왜곡하는 과장된 해석이므로 지양해야 합니다.
- 분석 과정에서 회계항등식에 따른 다중공선성을 식별하고 해소한 것은, 단순히 모델을 돌리는
  것을 넘어 **재무데이터의 구조적 특성을 이해하고 검증하는 과정**이 감사·재무분석에서 왜
  중요한지를 보여주는 사례입니다.

## 7. 한계 및 향후 개선 방향

- 표본 수(72건)가 적어 일반화에 한계가 있습니다. 소비재 외 업종이나 더 많은 연도로 확장하면
  더 신뢰도 높은 결과를 얻을 수 있습니다.
- 감사보수 데이터가 실제 DART 응답이 아닌 가상 샘플이므로, `dart_audit_fee_collector.py`로
  실제 데이터를 수집해 재검증이 필요합니다.
- 향후 감사위험 지표(예: 재고자산 비중, 매출채권 회전율 등)를 추가하면 감사보수 결정요인을
  더 폭넓게 분석할 수 있습니다.
